# v2.3 実機検証プレイブック + 実装ロードマップ

**Date:** 2026-04-10
**Version:** v2.3
**Goal:** 実機(VIVE Focus Vision)での全機能検証 → 実機依存機能実装 → v2.3リリース

---

## Overview

v2.2.0でソフトウェア側の実装が完了した。このプレイブックは実機検証から
v2.3リリースまでの全手順を5フェーズで定義する。

```
Phase 0 (30min)    → セットアップ
Phase 1 (2h)       → コア機能検証
Phase 2 (2h)       → v2.2新機能検証
Phase 3 (半日)     → パフォーマンス計測
Phase 4 (1週間)    → 実機依存機能実装 + v2.3リリース
```

各Phaseは前のPhaseの結果に依存する。Phase 1の映像ストリーミングが失敗したら
以降のPhaseには進まない。

---

## Phase 0: セットアップ (目標: 30分)

### 前提
- PC: Windows 10/11, NVIDIA GPU (GTX 1060+), SteamVR installed
- HMD: VIVE Focus Vision
- Network: PC と HMD が同一Wi-Fiネットワーク

### Step 0.1: PC側準備
- [ ] SteamVRインストール済み確認
- [ ] `cargo build --release -p focus-vision-companion`
- [ ] Companion App起動 → 「Install Driver」→ SteamVR再起動

### Step 0.2: HMD側準備
- [ ] 開発者モード有効化 (Settings → Developer → Developer mode ON)
- [ ] USB接続 → `adb devices` で認識確認
- [ ] Companion App → Deploy タブ → APKインストール

### Step 0.3: ネットワーク確認
- [ ] PC と HMD が同一Wi-Fiに接続されていることを確認
- [ ] PC のファイアウォールでTCP 9944, UDP 9945-9948を許可

### Step 0.4: 初回接続テスト
- [ ] SteamVR起動
- [ ] HMDでアプリ起動 → PIN入力画面表示確認
- [ ] Companion AppのPINをHMDに入力
- [ ] 「Connected」表示 → **Phase 0 PASS**

### 失敗時
```bash
adb logcat | grep -i "fvp\|focus"
```

## Phase 1: コア機能検証 (目標: 2時間)

**前提:** Phase 0 PASS（PIN接続成功）

### Step 1.1: 映像ストリーミング
- [ ] SteamVRホーム画面がHMDに表示される
- [ ] 頭の動きに追従する（レイテンシー体感チェック）
- [ ] 解像度が適切（ぼやけていない）
- [ ] フレームドロップなし（5分間連続）

記録: 体感レイテンシー `___`ms、体感品質 `良/可/不可`

**判定: FAIL → 停止。映像が動かないと他の検証は無意味。**

### Step 1.2: 音声
- [ ] PCのシステム音声がHMDで聞こえる
- [ ] 音声遅延が許容範囲（映像との同期）
- [ ] 音声途切れなし（3分間連続）

記録: 音声遅延体感 `___`ms

### Step 1.3: Face Tracking
- [ ] VRChat起動 → FT対応アバターロード
- [ ] 口の動き → HMDのブレンドシェイプ反映確認
- [ ] 目の動き → 視線追跡反映確認
- [ ] EMAスムージング動作確認（ジッター少ない）

記録: FT動作 `正常/部分的/非動作`

### Step 1.4: ハプティクス
- [ ] SteamVRのコントローラーテスト → HMD振動確認
- [ ] ゲーム内のハプティクスイベント → HMD振動確認

記録: 振動 `正常/弱い/なし`

### Step 1.5: 睡眠モード
- [ ] HMDを机に置いて5分待機 → 画面暗転確認
- [ ] HMDを持ち上げる → 即時復帰確認
- [ ] Companion Appのステータスで Sleep/Awake 切替確認

記録: 睡眠検出 `正常/過敏/無反応`

### Step 1.6: HMDダッシュボード
- [ ] VR内ダッシュボードを開く
- [ ] ビットレート変更 → 反映確認（Companion Appで値変化確認）
- [ ] レイテンシーウォーターフォール表示確認

記録: ダッシュボード `正常/部分的/非表示`

### Phase 1 判定
- Step 1.1 FAIL → **停止**
- Step 1.2-1.6 FAIL → 該当機能のバグとして記録、Phase 2に進む

## Phase 2: v2.2新機能検証 (目標: 2時間)

**前提:** Phase 1 Step 1.1 PASS

### Step 2.1: 適応FEC動作確認
- [ ] ストリーミング中のログで「AdaptiveFEC」メッセージ確認
- [ ] Wi-Fiを意図的に劣化（距離を離す/障害物）→ FEC%増加をログで確認
- [ ] Wi-Fi回復 → FEC%低下をログで確認

確認: ログで `AdaptiveFEC:` を grep

記録: FEC動作 `正常/変化なし/クラッシュ`

### Step 2.2: TCP再接続テスト
- [ ] ストリーミング中にHMDのWi-Fiを一時OFF → 5秒以内にON
- [ ] 自動再接続成功 → PIN再入力なしで復帰
- [ ] ストリーミング中にWi-Fi OFF → 10秒後にON → 新規接続（PIN再入力）

記録: 再接続 `5s以内復帰/新規接続OK/失敗`

> **注意:** 現在のhold logicはリスナー未保持（TODOS.md参照）。
> 5秒以内の復帰は期待通り動作しない可能性がある。Phase 4で修正予定。

### Step 2.3: セッションログ確認
- [ ] 10分ストリーミング
- [ ] `%APPDATA%/FocusVisionPCVR/sessions/` にJSONLファイル存在
- [ ] `cat session_*.jsonl | head` → JSON構造確認
- [ ] 複数レコード存在（10秒間隔 → 10分で~60レコード）

記録: ログ `正常/空/ファイルなし`

### Step 2.4: メモリ監視確認
- [ ] ストリーミング開始 → ログで「Memory monitor: baseline RSS = X MB」確認
- [ ] 30分後 → RSSが大幅増加していないことを確認

記録: ベースラインRSS `___`MB、30分後RSS `___`MB

### Step 2.5: Protocol v3ネゴシエーション
- [ ] ログで「protocol version: 3」を確認
- [ ] HMD側: `adb logcat | grep -i "protocol.*version"`

記録: ネゴシエーション `v3/v2/v1/不明`

### Phase 2 判定
- 全項目記録。FAILはバグとして次フェーズに持ち越し
- Step 2.2の再接続失敗は想定内

## Phase 4: 実機依存機能の実装 (目標: 1週間)
