## v1.0 スコープ内

### PIN ペアリングのレート制限追加
- **What:** 4桁PINペアリングに3回失敗で60秒ロックを実装
- **Why:** レート制限なしの4桁PIN（10,000通り）は数秒でブルートフォース可能。ローカルネットワーク前提でも共有環境（寮、オフィス等）では脆弱
- **Pros:** セキュリティの最低限保証。実装コストほぼゼロ
- **Cons:** なし
- **Context:** 外部レビューで指摘。ペアリング処理はRust Streaming Engineの接続管理モジュールで実装予定。3回失敗 → 60秒ロック → リセットの単純なカウンター
- **Depends on:** コントロールチャンネル（TCP）の基本実装

### オーディオパイプラインの仮想オーディオデバイス調査
- **What:** Windowsでの仮想オーディオデバイス作成方法を調査。VB-CABLE依存 vs カーネルドライバー自作 vs ALVRのオーディオドライバー流用
- **Why:** v1.1のオーディオパイプラインはWASAPI仮想オーディオデバイスを前提としているが、これにはカーネルモードドライバーが必要で巨大なスコープになる可能性。外部レビューで指摘
- **Pros:** v1.1開始前にスコープリスクを把握できる
- **Cons:** 調査に半日〜1日かかる
- **Context:** ALVRは独自のオーディオドライバー（alvr_audio）を実装している。VB-CABLEなどのサードパーティ依存は配布時のUX問題になる。調査対象: (1) ALVRのalvr_audioの構造と流用可能性 (2) Windows Audio Session API (WASAPI) loopback capture (3) VB-CABLE/VoiceMeeter依存の可否
- **Depends on:** なし（v1.0と並行して調査可能）
- **Blocked by:** なし
